## 데이터 분류 및 접근 로그

### 데이터 분류 기준

모든 데이터는 민감도에 따라 공개 데이터, 내부 데이터, 기밀 데이터로 분류해야 한다. 각 데이터 유형에 대한 접근 권한은 관련 부서의 승인을 받아야 하며, 기밀 데이터에 대한 접근은 최소한의 인원으로 제한한다.

### 접근 로그 기록

모든 데이터 접근은 로그로 기록되어야 하며, 로그에는 접근자 ID, 접근 시간, 접근한 데이터 유형이 포함되어야 한다. 로그는 자동으로 생성되며, 최소 1년 이상 보관해야 한다.

### 접근 권한 관리

데이터 접근 권한은 정기적으로 검토되어야 하며, 최소 분기마다 사용자의 필요에 따라 업데이트해야 한다. 권한 변경 사항은 즉시 로그에 기록하고 관련 부서에 통보한다.

### 접근 권한 승인 절차

신규 데이터 접근 요청은 반드시 문서화되어 관련 부서의 승인을 받아야 한다. 승인 절차는 3일 이내에 완료되어야 하며, 요청자는 승인 결과를 기록으로 남겨야 한다.

### 로그 감사

접근 로그는 매년 1회 이상 감사되어야 하며, 감사 결과는 문서로 기록하고 필요한 경우 적절한 조치를 취해야 한다. 감사 결과는 경영진에게 보고한다.

### 데이터 접근 교육

모든 직원은 데이터 접근 관련 교육을 연 1회 이상 이수해야 하며, 교육 내용은 데이터 분류 및 접근 로그의 중요성을 포함해야 한다. 교육 이수 후, 참석자는 교육 이수증을 제출해야 한다.

### 데이터 접근 모니터링

데이터 접근은 실시간으로 모니터링되어야 하며, 비정상적인 접근 시도를 탐지할 수 있는 시스템을 구축해야 한다. 이상 징후 발견 시 즉시 관련 부서에 통보하여 조치를 취한다.

### 데이터 접근 기록 삭제

불필요한 접근 로그는 데이터 보관 기간이 만료된 후 30일 이내에 안전하게 삭제해야 한다. 삭제 과정은 문서화되어야 하며, 삭제 기록은 최소 1년간 보관한다.

### 비인가 접근 대응 절차

비인가 접근이 발생한 경우, 즉시 시스템 관리자에게 보고하고 해당 접근을 차단해야 한다. 사건 발생 후 24시간 이내에 원인 분석 및 보고서를 작성한다.

### 접근 로그 보호

접근 로그 데이터는 암호화하여 저장하며, 로그 접근 권한은 최소한으로 제한해야 한다. 로그 데이터의 무결성을 보장하기 위한 정기적인 점검이 필요하다.

### 외부 감사 준비

외부 감사에 대비하여 모든 접근 로그는 정리되고 체계적으로 보관되어야 하며, 관련 자료는 언제든지 요청할 수 있도록 준비해야 한다. 감사 준비는 최소 1개월 전에 시작한다.

### 데이터 분류 및 로그 정책 검토

데이터 분류 및 접근 로그 정책은 매년 검토하여 최신 기준 및 규정에 부합해야 한다. 검토 결과는 문서화하여 관련 부서에 배포한다.

### 이해관계자 접근 통제

이해관계자에게 데이터 접근을 허용할 경우, 사전 승인을 받아야 하며, 접근 범위는 명확히 정의하여야 한다. 모든 이해관계자의 접근은 계약서에 명시되어야 한다.

### 데이터 접근 요청 처리

데이터 접근 요청은 5일 이내에 처리되어야 하며, 요청이 승인된 경우, 요청자에게 적절한 접근 방법을 안내해야 한다. 요청 거부 시 이유를 명확히 설명해야 한다.

### 변경 관리 절차

데이터 분류 체계나 접근 로그 관리 시스템의 변경은 반드시 문서화하고, 관련 부서의 승인을 받아야 한다. 이러한 변경 사항은 모든 직원에게 공지되어야 한다.
