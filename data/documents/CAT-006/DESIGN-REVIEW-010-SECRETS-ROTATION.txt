# 6. 개발 설계 보안성 검토

문서: 설계 보안성 검토 - 비밀·인증서 로테이션
용도: Bicep/인프라 설계 검토 시 RAG 참조

## 비밀·인증서 관리

### Key Vault 단일 소스

애플리케이션·인프라가 사용하는 비밀, 키, 인증서는 Key Vault에만 보관하고, Bicep·코드에는 Key Vault 참조(getSecret, reference 등)만 둔다. 여러 리소스에 동일 비밀을 복제하는 구성은 검토 시 통합을 요구한다.

### 로테이션 정책

사내 기준으로 중요 비밀·키는 90일 이내 로테이션을 권장한다. 설계도에 로테이션 주기 또는 자동 갱신(예: 인증서 자동 갱신)이 명시되어 있으면 가점이다. 영구 비밀(만료 없음)만 사용하는 설계는 검토 시 개선 권고한다.

### 관리형 ID 우선

VM·Web App·Function 등에서 외부 서비스 접근 시 가능한 한 Managed Identity를 사용하고, SP 비밀·계정 키 사용을 최소화한다. 설계 검토 시 identity 블록 부재 시 Managed Identity 도입을 권고한다.
