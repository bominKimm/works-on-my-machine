# 6. 개발 설계 보안성 검토

문서: 설계 보안성 검토 - RBAC·관리 평면
용도: Bicep/인프라 설계 검토 시 RAG 참조

## 관리 평면 접근 통제

### RBAC 최소 권한

구독·리소스 그룹·리소스에 부여하는 역할은 최소 권한 원칙에 따른다. Owner·Contributor는 운영·배포 파이프라인 전용 서비스 주체에만 제한하고, 개발자 계정에는 Reader 또는 특정 리소스만 Contributor로 제한한다. 설계도에 '*' 범위의 광역 권한 부여는 검토 시 수정 요구한다.

### 관리자 계정·서비스 주체

Bicep 내에서 관리자 로그인·비밀번호를 하드코딩하는 것은 금지이다. SQL administratorLoginPassword, VM adminPassword 등은 @secure() 파라미터로 받거나 Key Vault 참조만 사용한다. 서비스 주체(SP)는 리소스별로 분리하고, 단일 SP가 구독 전체 권한을 갖는 설계는 승인되지 않는다.

### 조건부 액세스·MFA

관리 포털·CLI 접근은 사내 조건부 액세스 정책과 MFA 의무 적용 대상이다. 인프라 설계에 MFA 우회·예외를 전제로 한 구성은 허용하지 않는다.
