# 6. 개발 설계 보안성 검토

문서: 설계 보안성 검토 - CI/CD·파이프라인 리소스
용도: Bicep/인프라 설계 검토 시 RAG 참조

## 파이프라인 인프라 보안

### 배포용 자격 증명

CI/CD에서 사용하는 서비스 주체·API 키는 파이프라인 전용으로 제한하고, 구독 Owner·Contributor를 넓게 부여하지 않는다. 배포 권한은 해당 리소스 그룹 또는 리소스 타입별로 최소 범위만 부여한다. 설계 검토 시 파이프라인 SP에 구독 전체 권한이 있는 경우 수정을 요구한다.

### 로그·아티팩트 보안

빌드·배포 로그에 비밀, 연결 문자열, 토큰이 출력되지 않도록 파이프라인 단계를 설계한다. 사내 정책에 따라 로그 마스킹·비밀 변수 사용을 의무화한다. Bicep 배포 로그가 공개 저장소에 노출되는 구성은 금지한다.

### 인프라 변경 승인

상용(Prod) 리소스 변경은 사내 승인 워크플로(예: PR 리뷰, 변경 관리 티켓)를 거치도록 설계한다. 자동 배포만 있고 승인 단계가 없는 Prod 파이프라인은 검토 시 보완을 권고한다.
