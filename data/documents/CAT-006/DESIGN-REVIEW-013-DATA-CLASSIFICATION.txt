# 6. 개발 설계 보안성 검토

문서: 설계 보안성 검토 - 데이터 분류·처리
용도: Bicep/인프라 설계 검토 시 RAG 참조

## 데이터 분류 반영

### 리소스별 분류 태그

민감도가 높은 데이터를 다루는 리소스(DB, Storage, API)에는 DataClassification 태그를 필수로 부여한다. confidential·restricted 등급 리소스는 암호화·접근 로그·네트워크 제한이 설계에 반드시 포함되어야 한다. 설계 검토 시 분류 태그 부재 시 보강을 요청한다.

### 개인정보 처리 리소스

개인정보를 저장·처리하는 리소스는 전사 개인정보 보호 정책에 따라 접근 제한, 암호화, 로그 보관, 반출 제한이 설계에 반영되어야 한다. Bicep에 PII·개인정보 관련 주석 또는 파라미터가 있으면 검토 시 해당 정책 준수 여부를 추가 확인한다.

### 크로스 보드 데이터

리전·국가를 넘어 데이터가 복제·처리되는 설계는 데이터 주권·개인정보 이전 정책을 만족해야 한다. 사내 기준으로 특정 등급 이상 데이터는 국내 리전만 사용하거나 이전 승인을 전제로 한다.
