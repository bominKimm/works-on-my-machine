# 6. 개발 설계 보안성 검토

문서: 설계 보안성 검토 - 감사·컴플라이언스
용도: Bicep/인프라 설계 검토 시 RAG 참조

## 감사 로그 의무

### 리소스별 감사 활성화

SQL 감사(Auditing), Key Vault 진단 로그, Storage 로깅, NSG 흐름 로그 등이 설계에 포함되어야 한다. 감사 비활성화는 예외 승인 없이 허용하지 않는다. 사내 기준으로 중요 리소스는 1년 이상 로그 보관이다.

### 변경 이력 추적

인프라 변경은 Azure Policy·Blueprint 또는 배포 이력으로 추적 가능해야 한다. Bicep·템플릿 버전 관리와 배포 파이프라인 로그를 설계에 전제하고, 누가 언제 어떤 리소스를 변경했는지 조회 가능한 구성을 권장한다.

## 컴플라이언스

### 규제 대응

금융·통신·공공 등 규제 대상 서비스는 해당 규제기관 요구사항(암호화, 접근 통제, 로그 등)이 설계에 반영되어야 한다. 설계 검토 시 규제 영역이 명시된 경우 해당 기준 준수 여부를 확인한다.

### Azure Policy 연동

구독 또는 리소스 그룹에 사내 Azure Policy 이니셔티브(예: TLS 1.2 의무, NSG 부착, 태그 의무)가 할당되는 설계를 권장한다. Bicep만으로 리소스를 배포하고 Policy가 없는 환경은 검토 시 Policy 도입을 안내한다.
