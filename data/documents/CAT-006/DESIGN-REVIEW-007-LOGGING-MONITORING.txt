# 6. 개발 설계 보안성 검토

문서: 설계 보안성 검토 - 로깅·모니터링
용도: Bicep/인프라 설계 검토 시 RAG 참조

## 로그 설계 기준

### 진단 설정 의무

스토리지, Key Vault, SQL, 웹앱, NSG 등 보안 관련 리소스는 Azure 진단 설정(diagnostic settings)을 통해 Log Analytics 또는 사내 지정 로그 수집소로 전송되도록 설계한다. 진단 로그 미활성화는 예외 승인 없이 허용하지 않는다.

### 로그 보관 기간

사내 정책에 따라 접근·감사·보안 로그는 최소 1년 이상 보관한다. 설계 시 retentionInDays(또는 동등 설정)가 365 미만이면 검토 시 상향 조정을 요구한다.

### 로그 무결성

로그는 무단 변경·삭제가 불가능한 방식으로 기록한다. 스토리지 계정의 불변 Blob, Log Analytics의 보존 정책 등을 활용하고, 개발자 권한만으로 로그를 지울 수 있는 구성은 금지한다.
