# 6. 개발 설계 보안성 검토

문서: 설계 보안성 검토 - App Service·Function
용도: Bicep/인프라 설계 검토 시 RAG 참조

## App Service 보안

### 인바운드 제한

웹앱은 가능한 한 VNet 통합 또는 Private Endpoint로만 접근되도록 설계한다. 인터넷 직접 노출이 불가피한 경우에도 IP 제한(accessRestriction) 또는 WAF를 반드시 적용한다. accessRestriction이 비어 있고 HTTPS만 적용한 설계는 검토 시 보완 요청한다.

### 관리 평면 잠금

App Service의 게시 프로필(FTP)·SCM 사이트 접근은 비활성화하거나 IP 제한을 둔다. ftpsState: Disabled, scmIpSecurityRestrictions 설정을 설계에 포함한다.

## Azure Functions

### HTTP 트리거 보안

Function App의 HTTP 트리거는 익명 호출을 허용하지 않고, Function Key 또는 AAD 인증을 사용하도록 설계한다. 설계도에 authLevel: anonymous만 있는 공용 Function은 검토 시 인증 적용을 요구한다.

### 스토리지 계정

Function App은 내부적으로 스토리지 계정을 사용한다. 해당 스토리지도 HTTPS 전용·TLS 1.2·방화벽 제한을 적용하고, 공개 액세스 차단을 원칙으로 한다.
