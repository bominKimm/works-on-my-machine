# 6. 개발 설계 보안성 검토

문서: 설계 보안성 검토 - 환경 분리
용도: Bicep/인프라 설계 검토 시 RAG 참조

## Dev/Staging/Prod 환경 분리

### 구독·리소스 그룹 분리

사내 기준으로 개발(Dev), 스테이징(Staging), 상용(Prod) 환경은 구독 또는 최소한 리소스 그룹 수준에서 분리한다. 동일 리소스 그룹 내에 prod와 dev 리소스가 혼재하는 설계는 검토 반려 사유가 된다.

### 네트워크 격리

Prod VNet은 Dev/Staging VNet과 피어링하지 않는다. 테스트용 데이터가 Prod로 유입되거나 Prod 자격 증명이 Dev에 노출되는 구성을 금지한다. 필요 시 관리용 점프 호스트만 제한적으로 연결한다.

### 상용망 정보 금지

Dev/Staging 코드·Bicep·설정에 상용망 IP, FQDN, 연결 문자열, API 키를 하드코딩하지 않는다. 환경변수 또는 파이프라인 시크릿 주입만 사용한다. 설계 검토 시 상용 리소스 식별자가 비프로덕션 템플릿에 포함되어 있으면 위반으로 지적한다.
